Lesson 2

DELETION COMMANDS

dw to delete a word when we are in the command mode.

d$ to delete to the end of the linewhen we are in the command mode.

command grammar
operator [count] motion

[x] x means the element of the set of parameters that can be put onto that location.

operator:

Now we just learn d operator for delete.

motion:

  • w - until the start or the next word, EXCLUDING its first character.
  • e - to the end of the current word, INCLUDING the last character.
  • $ - to the end of line, INCLUDING the last character.
    For example, d2w means delete 2 words from the cursor.

linux 命令的结构如下：

command [-option] [argument]

操作加选项加对象（参数），把命令理解成程序，想要运行程序，需要有参数和对象。其中，要知道，命令按照英文语法习惯，常常是谓+修饰+宾

再介绍一下我们经常用到的g++，这个都有些什么命令呢？

以gcc命令为例：(gcc short for GNU Compilation Collection)

$0.1 原理示范

When you invoke GCC, it normally does preprocessing, compilation,assembly and linking. The “overall options” allow you to stop this process at an intermediate stage.

上述内容说明GCC标准化（无论何种语言）编译为四个基本步骤

  • 预处理
  • 编译
  • 汇编
  • 链接

其中在命令中增加一些选项，可以让我们暂停在其中的任何一个中间阶段

命令格式：
g++ [option] [object] ([option] [object])? 其中()?代表一种表达语法的符号语言即(中出现的模式)0个或1个。

预处理

g++ -E filename.cpp
g++ -E filename.cpp -o filename.i

根据如图所示的情况可以得到以下几点结论

预处理的任务，将头文件宏定义等进行展开（可以考虑做一个预处理器）

编译

g++ -S filename.cpp | filename.i
g++ -S filename.cpp | filename.i -o filename.s

编译的任务就是形成汇编码

汇编

g++ -c filename.cpp | filename.i | filename.s
g++ -c filename.cpp | filename.i | filename.s -o filename.o

形成机器码（在c++框架下应该称为目标文件.obj）其中一个源文件将对应一个目标文件

链接

g++ filename.cpp | filename.i | filename.s | filename.o
g++ filename.cpp | filename.i | filename.s |filename.o -o filename.out

将多个源文件汇编得到的目标文件整合成一个二进制文件（注意目标文件和链接后得到的文件均为二进制文件）加入内容包括必要的初始化，系统库及其他组件以及启动代码等（均为二进制形式）——有关链接的更多内容等我后续学习吧

第二章

数学模型的引出

什么是数学模型

描述系统的输出变量与输入变量之间关系的数学表达式（成为我们建立数学模型过程中的初心）

我们为什么需要数学模型

以便定量给出系统中一些变量之间的相互关系,从而对控制系统进行各种分析和设计，包括稳定性和动态响应性能分析（如何根据数学模型分析动态响应和稳定性将成为后续的重点）

建立数学模型的两种方法

机理建模

  要求：了解全部原部件结构、对应的物理机理
  特点：准确，需要知道系统内部情况

实验建模

  要求：特殊输入信号辨识、采用模型逼近
  特点：不准确，不需要知道系统内部情况

建模要求
  1. 准确
  2. 简单
    简单的控制系统
    线性、定常、集中参数、单输入单输出控制系统

    微分方程模型

    核心公式

$$
a_n \frac{dc^n(t)}{dt^n} + a_{n-1} \frac{dc^{n-1}(t)}{dt^{n-1}} + \dots + a_{1} \frac{dc(t)}{dt} + a_0c(t) = b_n \frac{dr^n(t)}{dt^n} + b_{n-1} \frac{dr^{n-1}(t)}{dt^{n-1}} + \dots + b_{1} \frac{dr(t)}{dt} + b_0r(t)
$$

实际系统$n \geq m$且绝大多数$n \ge m$

微分方程建立的步骤

  • 系统原理线路图
  • 确定输入输出量
  • 建立各元件微分方程
  • 消去中间变量化简
  • 得I/O之间的微分方程
    由输入端开始，按照信号流向顺序列写方程

注意事项

  1. 线性化是针对工作点进行的，工作点决定了线性化方程的系数。因此
  2. 实际系统输入量变化范围较大，采用小偏差法建立线性模型会带来较大的误差
  3. 线性化后的微分方程通常是增量方程
  4. 这一讲的线性化方法，只能用于满足两个前提条件的非线性控制系统；间断点、折断点、非单值不可。（本质非线性）

第三章

线性系统时间响应的性能指标

最大超调量$\sigma_p$

响应曲线偏离稳态值的最大值，常以百分比表示，即最大百分比超调量$\sigma_p = \frac{c (t _{p})-c (\infty)}{c(\infty)}$

延滞时间$t _{d}$

响应曲线到达稳态值的50%所需的时间，称为延滞时间。

上升时间$t _{r}$

  1. 响应曲线从稳态值的10%到90%所需时间
  2. 响应曲线从稳态值的5%到95%所需时间
  3. 响应曲线从0开始第一次到达稳态值所需的时间

Remark：一般对有振荡的系统常用c，对无振荡的系统常用a

峰值时间$t _{p}$

响应曲线到达第一个峰值所需的时间，定义为峰值时间。

调整时间$t _{s}$

响应曲线从0开始到进入稳态值的95%105%(或98%102%)误差带所需的时间，定义为调整时间。
